# 完整标准规范

> 本文档包含定时任务标准的完整规范

---

## 一、执行标准

### 1.1 任务理解与确认

| 要求 | 说明 |
|------|------|
| **严格按任务要求执行** | 不添加任务描述中未提及的内容 |
| **模糊要求处理** | 遇到以下情况视为模糊，需在任务设定时与用户确认：<br>- 输出格式未明确（如字数、段落结构）<br>- 数据来源未指定（如使用哪个 API/网站）<br>- 时间范围未定义（如"最近"是多久）<br>- 推送目标未指定（Discord Channel ID） |

### 1.2 任务修改原则

| 规则 | 说明 |
|------|------|
| **针对性修改** | 用户要求完善任务时，只修改用户指定的部分 |
| **保护现有内容** | 不修改任务中其他已确认的描述 |
| **修改记录** | 记录修改时间和修改内容摘要 |

### 1.3 输出流程

```
数据采集 → 存储到 PostgreSQL → 推送到 Discord Channel
```

| 步骤 | 要求 |
|------|------|
| **数据存储** | 先将输出内容存储到数据库 |
| **推送 Discord** | 从数据库读取内容，推送到指定 Channel |
| **输出限制** | 推送后不在当前会话输出其他内容 |

### 1.4 输出内容规范

| 允许 | 禁止 |
|------|------|
| ✅ 任务要求的数据内容 | ❌ "任务已完成"等状态信息 |
| ✅ 结构化的数据展示 | ❌ 调试日志或中间过程 |
| ✅ 必要的数据来源标注 | ❌ 未经要求的总结或评价 |

## 二、校验标准

### 2.1 自动校验流程

```
新任务/任务更新 → 执行一次任务 → 检查是否符合标准 → [不符合] → 分析原因 → 完善任务 → 重复校验
                                                           ↓ [符合]
                                                      通知用户
```

### 2.2 校验清单

| # | 检查项 | 通过标准 | 失败处理 |
|---|--------|---------|---------|
| 1 | **任务理解** | 没有模糊的要求 | 与用户确认 |
| 2 | **数据来源** | 来源明确且可访问 | 明确数据来源 |
| 3 | **数据存储** | 成功写入数据库 | 检查数据库连接 |
| 4 | **推送 Discord** | 成功推送到指定 Channel | 检查 Channel ID |
| 5 | **输出限制** | 没有多余输出 | 调整输出逻辑 |
| 6 | **内容质量** | 符合任务要求的内容 | 调整内容生成 |

### 2.3 循环限制

| 参数 | 值 | 说明 |
|------|-----|------|
| **最大循环次数** | 3 次 | 超过后停止并通知用户 |
| **每次间隔** | 5 分钟 | 避免频繁执行 |
| **停止条件** | 校验通过 或 循环3次 | 任一条件满足即停止 |
| **单次执行超时** | 10 分钟 | 单次任务执行超过10分钟视为失败 |

### 2.4 通知格式

#### 校验通过
```
✅ 任务校验通过

**任务名称**: [任务名]
**校验时间**: [时间]
**执行结果**: 符合所有标准
```

#### 校验失败（3次后）
```
❌ 任务校验失败

**任务名称**: [任务名]
**失败原因**: [具体原因]
**建议修改**: [修改建议]
```

## 三、配置规范

### 3.1 Discord Channel 配置

| 场景 | Channel ID 来源 |
|------|----------------|
| 任务描述中指定 | 使用任务描述中的 ID |
| 任务描述中未指定 | 询问用户 |
| 用户后续指定 | 更新任务描述 |

**获取方式**：
1. 在 Discord 频道右键 → 「复制 ID」
2. 或者使用 `openclaw status` 查看当前频道 ID

### 3.2 数据库配置

> ⚠️ 数据库配置由每个任务单独定义

每个任务应在任务描述中指定：
- 数据库类型（如 PostgreSQL）
- 表名和字段结构
- 存储格式要求

## 四、任务描述模板

### 4.1 标准模板

```markdown
# [任务名称]

## 基本信息

| 项目 | 值 |
|------|-----|
| **任务名称** | [任务名称] |
| **执行时间** | [cron 表达式或自然语言描述] (时区) |
| **推送目标** | Discord Channel ID: [ID] |
| **创建时间** | [YYYY-MM-DD] |
| **最后更新** | [YYYY-MM-DD] |

## 数据来源

[描述数据来源，如 API、网站、数据库等]

- 来源 1: [URL 或描述]
- 来源 2: [URL 或描述]

## 输出要求

| 项目 | 要求 |
|------|------|
| **格式** | [Markdown/JSON/纯文本] |
| **字数限制** | [每条/总数] |
| **条目数量** | [TOP N] |
| **其他** | [其他要求] |

## 存储配置

| 项目 | 值 |
|------|-----|
| **数据库** | [类型] ([连接信息]) |
| **表名** | [表名] |
| **字段** | [字段列表] |

## 执行逻辑

[描述任务的具体执行步骤，如需要]

## 变更记录

| 日期 | 变更内容 |
|------|---------|
| [日期] | [变更描述] |
```

### 4.2 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| **任务名称** | ✅ | 唯一标识任务的名称 |
| **执行时间** | ✅ | cron 表达式或自然语言（必须包含时区） |
| **推送目标** | ✅ | Discord Channel ID |
| **数据来源** | ✅ | 数据获取的来源 |
| **输出要求** | ✅ | 输出格式、字数、条目数等 |
| **存储配置** | ❌ | 如需持久化则填写 |
| **执行逻辑** | ❌ | 复杂任务需要填写 |
| **变更记录** | ❌ | 更新任务时填写 |

### 4.3 格式校验

新增或更新任务时，自动检查以下格式要求：

| # | 检查项 | 要求 |
|---|--------|------|
| 1 | **标题层级** | 使用 `#` 作为任务名称，`##` 作为章节 |
| 2 | **基本信息表格** | 必须包含表格格式的基本信息 |
| 3 | **时区标注** | 执行时间必须包含时区（如 Asia/Shanghai） |
| 4 | **Channel ID 格式** | 必须是数字 ID，不是频道名称 |
| 5 | **列表格式** | 数据来源使用 Markdown 列表 |

## 五、效果评估与复审 (SMART-ER)

### 5.1 效果评估

每次任务执行后，自动评估以下指标：

| 指标 | 衡量标准 | 目标值 |
|------|---------|--------|
| **执行成功率** | 成功次数 / 总执行次数 | ≥ 95% |
| **推送成功率** | Discord 推送成功次数 / 总执行次数 | ≥ 95% |
| **内容质量** | 符合任务要求的执行次数 / 总执行次数 | 100% |
| **执行时长** | 任务执行耗时 | ≤ 5 分钟 |

**异常处理**：
- 连续 3 次执行失败 → 自动通知用户
- 执行成功率 < 80% → 触发任务复审

### 5.2 定期复审

| 复审周期 | 触发条件 | 复审内容 |
|---------|---------|---------|
| **每周** | 自动触发 | 任务执行效果、数据来源有效性 |
| **按需** | 用户要求 | 任务标准是否需要调整 |
| **异常触发** | 连续失败 3 次 | 任务描述是否需要修改 |

**复审流程**：
```
收集执行数据 → 分析效果指标 → 识别问题 → 提出改进建议 → 通知用户
```
