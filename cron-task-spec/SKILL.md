# 定时任务标准规范

> 基于 **SMART-ER** 原则：Specific, Measurable, Achievable, Relevant, Time-bound, Evaluated, Reviewed

---

## 核心原则

1. **严格按任务要求执行** - 不添加任务描述中未提及的内容
2. **模糊要求需确认** - 以下情况视为模糊，需与用户确认：
   - 输出格式未明确
   - 数据来源未指定
   - 时间范围未定义
   - 推送目标未指定

3. **针对性修改** - 只修改用户指定的部分，保护其他内容

---

## 输出流程

```
数据采集 → 存储到数据库 → 推送到 Discord Channel
```

**推送后不在当前会话输出其他内容（如"任务已完成"等）**

---

## 校验流程

```
新任务/更新 → 执行一次 → 检查标准 → [不通过] → 分析原因 → 完善任务 → 重复（最多3次）
                                         ↓ [通过]
                                    通知用户
```

### 校验清单

| # | 检查项 | 失败处理 |
|---|--------|---------|
| 1 | 任务理解明确 | 与用户确认 |
| 2 | 数据来源可访问 | 明确来源 |
| 3 | 数据存储成功 | 检查连接 |
| 4 | Discord 推送成功 | 检查 ID |
| 5 | 没有多余输出 | 调整逻辑 |
| 6 | 内容质量达标 | 调整生成 |

### 循环限制

- **最大循环次数**: 3 次
- **每次间隔**: 5 分钟
- **单次超时**: 10 分钟

## 任务描述模板

> ⚠️ 所有任务必须使用统一格式

### 必填字段

| 字段 | 说明 |
|------|------|
| **任务名称** | 唯一标识 |
| **执行时间** | cron 表达式 + 时区 |
| **推送目标** | Discord Channel ID（数字） |
| **数据来源** | Markdown 列表 |
| **输出要求** | 格式、字数、条目数 |

### 可选字段

| 字段 | 说明 |
|------|------|
| 存储配置 | 数据库类型、表名、字段 |
| 执行逻辑 | 复杂任务的执行步骤 |
| 变更记录 | 更新时填写 |

### 模板示例

```markdown
# [任务名称]

## 基本信息

| 项目 | 值 |
|------|-----|
| **任务名称** | [名称] |
| **执行时间** | [cron] (Asia/Shanghai) |
| **推送目标** | Discord Channel ID: [数字ID] |
| **创建时间** | [YYYY-MM-DD] |

## 数据来源

- 来源1: [URL]
- 来源2: [URL]

## 输出要求

| 项目 | 要求 |
|------|------|
| **格式** | Markdown/JSON/文本 |
| **条目数** | TOP N |
| **字数** | 每条 X 字 |

## 存储配置（可选）

- 数据库: PostgreSQL (localhost:5432/db)
- 表名: [表名]
- 字段: [字段列表]
```

## 效果评估

每次执行后自动评估：

| 指标 | 目标值 |
|------|--------|
| 执行成功率 | ≥ 95% |
| 推送成功率 | ≥ 95% |
| 执行时长 | ≤ 5 分钟 |

**异常处理**：
- 连续 3 次失败 → 通知用户
- 成功率 < 80% → 触发复审

创建/更新任务时，自动检查：

- [ ] 标题使用 `#`，章节使用 `##`
- [ ] 基本信息包含表格
- [ ] 执行时间包含时区
- [ ] Channel ID 是数字
- [ ] 数据来源是列表格式
